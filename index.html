HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The style element defines the CSS styles associated with a website.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • HTML handles the structure and content of a webpage.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS is used to define how webpage content is presented to users such as colors, aesthetics, visual layout and organization.
  • Inline Styles are more permanant and located in specific HTML tags.
  • Internal Styles contain rules for the CSS in the head of the page used to set classes and IDs.
  • External Styles are most common and allow the CSS files to be separate from the rest of the code.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin branch-name: pulls code from that branch
  • git push origin branch-name: pushes code from that branch
  • git add -A: allows you to add all changes to the current branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • If and For loops can be set as functions to make them more durable and reusable.
  • You have to call a function after it is set in order for it to run.
  • Variables can be set as string, numbers, boolean (true/false) and many more.
  • Variables are case sensitive.
  • Strings must be contained in quotation marks unlike numbers or Boolean values.
  • Arrays are useful tool for storing multiple items with differnt data types, but it doesn't have to have multiple types.
